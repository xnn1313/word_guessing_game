# 猜词游戏微信小程序（开发版）

这是现有脑力游戏的原生微信小程序客户端，复用同一套 Flask API、账号、SQLite 存档、
猜词分类关卡和双人房间，并增加数独、成语填字与记忆翻牌。当前开发版使用用户名/密码
和 Bearer Token，可以先在微信开发者工具中联调。

## 已实现

- 首页、账号登录/注册和个人进度
- 经典猜词、相似度/时间排序、查看答案和重置
- 13 个分类、260 关、解锁、星级和下一关
- 双人创建/加入房间、实时比分、倒计时、猜测记录和双方再战
- 每日/练习数独：三种难度、候选数、擦除、错误格、提示和结果页
- 成语填字：60 个普通关卡、每日题、本地/云端进度、提示和连续闯关
- 记忆翻牌：三种难度、三套牌面、每日/练习模式和自动结算
- 游戏大厅动态汇总；游客本地续玩，登录用户从云端恢复
- Token 哈希存储，网页 Cookie 登录保持兼容

## 在微信开发者工具中打开

1. 在电脑上安装微信开发者工具。
2. 导入项目目录 `word_guessing_game/miniprogram`。
3. 使用自己的测试 AppID 时，在开发者工具中替换 `project.config.json` 里的 AppID。
4. 打开“详情 → 本地设置”，勾选“不校验合法域名、web-view、TLS 版本以及 HTTPS 证书”。
5. 如需连接其他后台，修改 `config/index.ts` 中的 `API_BASE`：

   ```ts
   export const API_BASE = "http://你的服务器IP:5000/api";
   ```

   `127.0.0.1` 指的是运行微信开发者工具的电脑，不是远程服务器。若开发者工具和后端不在
   同一台机器，必须填写电脑可以访问到的服务器地址。

6. 确保后端服务已启动，并能访问 `/api/games/overview`，然后点击“编译”。

## 本地检查

```bash
cd miniprogram
npm install
npm run check
```

## 开发阶段的安全提醒

用户名和密码不应通过公网 HTTP 传输。如果从外网联调，请至少通过 VPN/SSH 隧道访问，
或提前给测试域名配置 HTTPS。正式发布时还需换成合法 HTTPS 域名，并接入 `wx.login`；
AppSecret 只能放在服务器环境变量中。

## 当前限制

- 游客存档保留在当前设备，登录后使用云端存档，两个来源暂不自动合并。
- 双人状态沿用现有每秒轮询，后续可升级为 WebSocket + Redis。
- 好友分享和两台真机联调需要可用的小程序 AppID，并配置服务器域名。
- Linux 服务器不能运行官方微信开发者工具，最终视觉和真机交互需要在 Windows/macOS 工具中确认。
